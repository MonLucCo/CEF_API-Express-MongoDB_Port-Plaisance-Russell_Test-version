# ests de niveau‑3 : Tests E2E

Les tests E2E valident l’API complète dans un environnement réel.

## Objectifs

- Vérifier le comportement global de l’API  
- Tester les scénarios complets utilisateur  
- Valider la cohérence entre les différentes couches  
- Préparer la livraison finale

## Outils

- **Postman**  
- Base MongoDB réelle  
- Serveur Express réel

## Scénarios typiques

- Inscription utilisateur  
- Connexion utilisateur  
- Suppression utilisateur  
- Accès à une route protégée  
- Gestion des erreurs
