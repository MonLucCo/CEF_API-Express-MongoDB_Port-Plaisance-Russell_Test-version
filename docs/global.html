authMiddleware(req, res, next) → {Object|Object|void}

Middleware d'authentification JWT. Vérifie la présence et la validité du token transmis dans l'en-tête HTTP `Authorization: Bearer `, ou dans les cookies. La priorité est donnée au token dans les cookies, puis à celui dans les headers. Comportements possibles : - 401 : token manquant ou mal formé - 401 : token invalide (JsonWebTokenError) - 401 : token expiré (TokenExpiredError) - 500 : erreur interne lors de la vérification du token - next() : token valide → `req.userId` défini Effets de bord : - Ajoute `req.userId` contenant l'identifiant utilisateur extrait du token.
Parameters:
Name Type Description
req Object Requête Express
res Object Réponse Express
next function Fonction permettant de passer au middleware suivant
Version:

Reservation

Modèle Mongoose représentant une réservation d’un catway du port de plaisance Russell. Une réservation associe un client, un bateau et une période de séjour à un catway existant.
